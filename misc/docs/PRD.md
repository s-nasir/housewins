## 1. Problem Statement

The general public has a poor intuitive understanding of casino game odds. Real casinos are designed to obscure the mathematical disadvantage players face. There is no accessible, free, interactive platform that lets users experience casino games while transparently educating them on the house edge and real-world odds.

---

## 2. Solution

A free, browser-based casino simulation website where:
- Users play 6 authentic casino games using fictional in-game currency
- No real money is involved at any point
- The mathematical house edge is exposed and visualized per game
- A live global counter shows aggregate house profit and player winnings
- Currency is freely provided via a passive earn system
- The experience is immersive enough to demonstrate the psychological pull of gambling

---

## 3. Core Constraints

- **No user accounts, no login, no registration**
- **No real money** — all currency is fictional and non-withdrawable
- **No user data stored server-side** — session state lives in localStorage/sessionStorage only
- **All code generated by AI** — architecture must support clean AI codegen (deep modules, clear interfaces)
- **Free hosting only** — all infrastructure must run on free tiers

---

## 4. Users & User Stories

### Primary User: Curious Player
A person who wants to experience casino games without financial risk, or who wants to understand why "the house always wins."

**User Stories:**
1. As a new visitor, I receive a welcome currency pack so I can play immediately without waiting.
2. As a player, I earn passive currency while the site is open so I never get permanently stuck.
3. As a player, I can see my current Chips, Tokens, and Tickets balance at all times.
4. As a player, I can navigate to any of the 6 game pages from the main lobby.
5. As a player, I can see the exact odds and house edge for every bet I place.
6. As a player on Roulette, I can place chips on any bet position on a 2D table and watch the wheel spin.
7. As a player on Blackjack, I can Hit, Stand, Double, or Split and play against the house dealer.
8. As a player on Scratchers, I can purchase a ticket and scratch the coating with my mouse/touch.
9. As a player on Slots, I can set my bet per spin and watch 5 reels spin with win animations.
10. As a player on Keno, I can pick up to 10 numbers from a grid of 80 and watch 20 numbers be drawn.
11. As a player on Video Poker, I can hold cards and draw replacements to build the best hand.
12. As a player, I can see the global house profit counter and total player winnings update in real time.
13. As a player, I can see my personal session stats (won/lost this session).
14. As a visitor from outside the US, I see the global counters displayed in my local currency.
15. As a first-time visitor, I see a disclaimer popup informing me this is a simulation.
16. As a player who runs out of currency, I can wait for the passive drip to refill my balance.
17. As a supporter, I can visit a support page and contribute to the developer via Ko-fi.
18. As a player, I can mute all audio with a single toggle.

## 6. Feature Specifications

### 6.1 Global Layout
- **Header (all pages):** HouseWins.gg logo, live global counter (House Profit | Player Winnings), currency balance display (Chips / Tokens / Tickets), mute toggle
- **Footer (lobby only):** Disclaimer link, Responsible Gambling link, Ko-fi support link, Terms popup trigger
- **Disclaimer Popup:** Shown once per new session on first visit. Dismissible. Re-accessible from footer.

### 6.2 Currency System
- **Chips:** Primary play currency (Roulette, Blackjack)
- **Tokens:** Secondary currency (Slots, Keno, Video Poker)
- **Tickets:** High-value currency (Scratchers input)
- **Starting pack (new session):** 1,000 Chips, 500 Tokens, 10 Tickets
- **Passive earn:** 10 Chips + 5 Tokens every 10 seconds while site is open
  - Rate tuning target: adjust to 15 Chips + 5 Tokens per 20–30 seconds based on average session play time data
  - Tab can be backgrounded — no focus requirement
  - Only ONE tab earns per browser (BroadcastChannel API enforcement)
- **Session isolation:** Each browser = one session. Different browser = different session.
- **Storage:** Zustand store + localStorage. Lost on browser data clear.

### 6.3 Wager Ranges Per Game

| Game | Currency | Min | Max |
|------|----------|-----|-----|
| Roulette | Chips | 5 | 500 |
| Blackjack | Chips | 10 | 1,000 |
| Scratchers | Tickets | 1 | 5 |
| Slots | Tokens | 1 | 100 |
| Keno | Tokens | 5 | 200 |
| Video Poker | Tokens | 5 | 250 |

### 6.4 Global Counter System
- **GlobalWagers:** Sum of all bet amounts placed by all users, all time
- **GlobalWinnings:** Sum of all payouts distributed, all time
- **GrossProfit:** GlobalWagers − GlobalWinnings
- **NetHouseProfit:** GrossProfit × 0.85 (15% simulated tax)
- **Storage:** Upstash Redis (atomic INCRBYFLOAT commands)
- **Delivery:** SSE stream from Cloudflare Worker → all connected clients
- **Counter starts in USD.** On page load, CF-IPCountry detects user country, converts display amount using ExchangeRate-API.

### 6.5 RNG System
- All game outcomes determined server-side via `crypto.randomInt()` in Cloudflare Worker
- Client sends bet(s) → Worker runs RNG + house math → returns outcome + updated counters
- Provably fair: each round, Worker commits SHA-256(seed) before resolution, reveals seed after
- Odds visualization panel on each game page: displays win probability, house edge, expected loss per 100 units wagered

### 6.6 Game Engines

#### American Roulette
- 38 slots (1–36 + 0 + 00), House Edge: 5.26%
- Accepts array of bet positions + amounts
- Stateless: one request per spin
- 2D animated wheel (Canvas/CSS)
- All inside and outside bet types supported

#### Classic Blackjack
- 6-deck shoe, House Edge: ~0.5%
- State machine: Deal → Player Action → Dealer Resolution
- Hand state tracked in sessionStorage (prevents refresh abuse)
- Actions: Hit, Stand, Double, Split

#### Digital Scratchers
- Pre-generated result weighted distribution on Worker
- Canvas scratch effect (mouse + touch)
- House Edge: 15–25%

#### Video Slots (5-reel, 20 fixed paylines)
- Weighted reel stops (PAR sheet)
- RTP: ~92–95% (House Edge: 5–8%)
- Win animations, near-miss effect
- GSAP reel spin animation

#### Keno
- 80-number grid, player picks 1–10
- Worker draws 20 using Fisher-Yates shuffle
- House Edge: 20–30%
- Hypergeometric payout table

#### Video Poker (Jacks or Better)
- 52-card deck, 5 cards dealt
- Hold/Discard mechanic
- Hand evaluator: Royal Flush → Pair of Jacks+
- Full Pay 9/6 table, House Edge: 0.46%

## 8. Technical Architecture (Summary)

See `ADR.md` for full decision rationale.

```
Browser (Vite + React SPA)
  │
  ├── Zustand store ←→ localStorage (session state)
  ├── SSE connection → Cloudflare Worker (live counter updates)
  └── HTTP requests → Cloudflare Workers (game engine API)
                          │
                          ├── crypto.randomInt() (RNG)
                          ├── Game engine logic (pure functions)
                          ├── CF-IPCountry header (geo detection)
                          └── Upstash Redis (INCR global counters)
```

## 9. Out of Scope (v1.0)

The following are explicitly NOT part of v1.0. Do not implement these:
- User accounts, profiles, leaderboards
- Real money transactions (Stripe integration is post-launch)
- 3D graphics / Three.js
- Multiplayer / shared game tables
- Mobile app (iOS/Android)
- Progressive Web App (PWA) features
- Email notifications
- Chat / social features
- Additional games beyond the 6 defined
- Admin dashboard
- Analytics/tracking beyond the two global counters
